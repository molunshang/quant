# ETF 筛选与数据清洗管道（EtfFilterPipeline）

ETF 双动量策略能够长期有效的前提，是构建一个"相关性低、宽基为主、具备强流动性"的黄金种子池。

全市场 ETF 约上千只，如果将全部标的直接投入动量计算，会出现三个致命问题：

1. **流动性黑天鹅** — 某些主题 ETF 涨幅巨大、动量极强，但日均成交额仅几百万，100 万资金进出会产生巨大冲击成本
2. **影子共振** — 沪深300ETF、上证50ETF、沪深300成长ETF 高度相关，它们会同时霸榜动量排名，导致组合表面分散实则集中
3. **清盘风险** — 规模过小的 ETF（< 5 亿）面临清盘，一旦发生仓位将被强制平仓

本管道设计为独立于具体策略的预处理模块，在动量计算前对全市场 ETF 执行三层硬性筛选。

---

## 1. 漏斗模型：三层硬性筛选

```
全市场 ETF (~1000只)
        │
        ▼
┌─────────────────────────┐
│ 第一层: 规模筛选        │  剔除 AUM < 5 亿
│ (Size Filter)           │  → 剩余 ~400 只
└───────────┬─────────────┘
            ▼
┌─────────────────────────┐
│ 第二层: 流动性筛选      │  剔除 20日均成交额 < 5000万
│ (Liquidity Filter)      │  → 剩余 ~100 只
└───────────┬─────────────┘
            ▼
┌─────────────────────────┐
│ 第三层: 折溢价筛选      │  剔除 |折溢价率| > 1%
│ (Tracking/Premium)      │  → 剩余 ~80 只
└───────────┬─────────────┘
            ▼
┌─────────────────────────┐
│ 第四层: 大类资产去重    │  每类只保留流动性最强的 1 只
│ (Asset Class Dedup)     │  → 最终种子池 ~5-8 只
└─────────────────────────┘
```

### 1.1 规模筛选（Size Filter）

- **硬性门槛**：资产净值（AUM）> 5 亿元
- **数据来源**：掘金 `get_symbols()` 返回的日频数据 + 行情估算，或 `fnd_get_etf_constituents()` 推算规模
- **剔除原因**：规模 < 5 亿的 ETF 存在清盘风险，且容易被少量资金控盘操纵价格

### 1.2 流动性筛选（Liquidity Filter）

- **硬性门槛**：过去 20 个交易日日均成交额 > 5000 万元
- **数据来源**：掘金 `history(frequency='1d', fields='amount')` 获取每日成交额
- **剔除原因**：个人 100 万级别资金如果快进快出（特别是触发空仓风控时），必须有足够的流动性支撑。日均成交额低于 5000 万意味着单次买卖会产生巨大的冲击成本和滑点

### 1.3 折溢价筛选（Tracking & Premium）

- **硬性门槛**：|折溢价率| ≤ 1%
- **数据来源**：实时模式下通过 tick 数据的 `iopv`（基金份额参考净值）计算，公式为 `premium_rate = (price - iopv) / iopv`
- **剔除原因**：折溢价率过大的 ETF 说明二级市场价格与净值严重偏离，可能是流动性枯竭或市场操纵的信号，不适合作为动量交易标的
- **回测模式**：折溢价数据在回测中不可用（tick 数据不含 iopv），该层默认跳过

### 1.4 大类资产去重（Asset Class Dedup）

动量策略的核心竞争力来自"资产轮动"——在不同大类之间切换，而非在同一类资产内部追涨。若沪深300、上证50、沪深300成长这三只高度相关的 ETF 同时入选，动量排名时它们会因同涨同跌而霸占全部席位，导致组合承受三倍的系统性风险。

**去重规则**：同一大类资产只保留流动性最强（20 日均成交额最大）的一只。

**预设大类映射表**：

| 大类 | 说明 | 代表标的 |
|------|------|---------|
| A 股大盘 | 沪深 300、上证 50 | `SHSE.510300`, `SHSE.510050` |
| A 股中小盘 | 中证 500/1000/2000 | `SHSE.510500`, `SHSE.512100`, `SHSE.515080` |
| 红利防御 | 红利低波、中证红利 | `SHSE.515100`, `SHSE.510880` |
| 跨境对冲 | 纳指 ETF、标普 500 ETF | `SHSE.513100`, `SHSE.513500` |
| 商品避险 | 黄金 ETF、豆粕 ETF | `SHSE.518880`, `SHSE.518850` |

## 2. 数据来源（掘金 SDK）

| 筛选层 | 所需数据 | SDK 函数 | 说明 |
|--------|---------|---------|------|
| 规模 | AUM / 资产净值 | `get_symbols()` + `history()` 估算 | 免费函数不直接提供 AUM，可通过净值×份额估算；实盘中也可跳过硬 AUM 检查，以流动性为主 |
| 流动性 | 20 日日均成交额 | `history(frequency='1d', fields='amount')` | 批量拉取全市场 ETF 的日线成交额 |
| 折溢价 | IOPV vs 现价 | tick 数据 `iopv` 字段 / `last_tick()` | 仅实时模式可用，回测模式跳过 |
| 分类去重 | ETF 基础信息 | `get_symbols()` / `get_symbol_infos()` | 获取标的名称、交易所、板块分类 |
| 全池 | 全部 ETF 代码 | `get_symbols(sec_type1=1020, sec_type2=102001)` | 拉取全市场股票型 ETF 列表 |

## 3. 管道对接策略的完整闭环

```
定时触发（每日盘前/尾盘）
      │
      ▼
┌──────────────────────────────────┐
│ Step A: 获取全市场 ETF 数据       │
│ get_symbols(sec_type1=1020, ...)  │ → 全市场 ETF 列表
│ history(frequency='1d', ...)      │ → 20 日成交额数据
└───────────────┬──────────────────┘
                ▼
┌──────────────────────────────────┐
│ Step B: EtfFilterPipeline 执行筛选 │
│ filter_and_select_seeds(a, b)     │ → 健康种子池 (5-8 只)
└───────────────┬──────────────────┘
                ▼
┌──────────────────────────────────┐
│ Step C: 动量引擎计算              │
│ EtfMomentumGridStrategy           │
│ calculate_trading_signals(...)    │ → 目标仓位
└───────────────┬──────────────────┘
                ▼
┌──────────────────────────────────┐
│ Step D: 执行交易                  │
│ order_target_value(...)           │ → 差额调仓
└──────────────────────────────────┘
```

## 4. 实战价值

1. **防止流动性黑天鹅** — A 股很多特定主题 ETF（低碳、传媒等）看似涨幅巨大、动量极强，但日均成交额可能只有几百万。通过强制流动性洗礼，100 万资金永远不会误入这些容易"买进去就出不来"的流动性死水。

2. **拒绝影子共振** — 通过大类资产分类去重，确保入选的标的在底层逻辑上天然对冲。这是资产轮动策略控制回撤的底层硬逻辑——不是靠止损，而是靠"不把鸡蛋放在同一个篮子里"。

3. **适应市场变化** — ETF 市场不断有新标的上市、老标的退市或规模萎缩。每日动态筛选确保种子池始终反映最新的市场结构，而非依赖一成不变的预设列表。
