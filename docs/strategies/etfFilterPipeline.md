# ETF 筛选与数据清洗管道（EtfFilterPipeline）

ETF 双动量策略能够长期有效的前提，是构建一个"相关性低、宽基为主、具备强流动性"的黄金种子池。

全市场 ETF 约上千只，如果将全部标的直接投入动量计算，会出现三个致命问题：

1. **流动性黑天鹅** — 某些主题 ETF 涨幅巨大、动量极强，但日均成交额仅几百万，100 万资金进出会产生巨大冲击成本
2. **影子共振** — 沪深300ETF、上证50ETF、沪深300成长ETF 高度相关，它们会同时霸榜动量排名，导致组合表面分散实则集中
3. **清盘风险** — 日均成交额萎靡的 ETF 通常规模也极小，面临清盘，一旦发生仓位将被强制平仓

本管道设计为独立于具体策略的预处理模块，在动量计算前对全市场 ETF 执行四层筛选。**所有筛选层均基于掘金免费 SDK 可获取的数据。**

---

## 1. 漏斗模型：四层筛选

```
全市场 ETF (~1000只)
        │
        ▼
┌─────────────────────────┐
│ 第 1 层: 交易状态筛选   │  剔除停牌、临近退市、数据不足
│ (Trading Status)        │  数据: get_symbols() → is_suspended
│                         │  → 剩余 ~700 只
└───────────┬─────────────┘
            ▼
┌─────────────────────────┐
│ 第 2 层: 流动性筛选     │  剔除 20日均成交额 < 5000万
│ (Liquidity)             │  数据: history(fields='amount')
│                         │  → 剩余 ~100 只
│                         │  ★ 同时作为规模代理
└───────────┬─────────────┘
            ▼
┌─────────────────────────┐
│ 第 3 层: 波动率质量     │  剔除 年化波动率 > 40%
│ (Volatility Quality)    │  剔除 单日涨跌幅 > 15%
│                         │  数据: history(fields='close')
│                         │  → 剩余 ~80 只
└───────────┬─────────────┘
            ▼
┌─────────────────────────┐
│ 第 4 层: 大类资产去重   │  每类只保留流动性最强的 1 只
│ (Asset Class Dedup)     │  → 最终种子池 ~5-8 只
└─────────────────────────┘
```

## 2. 各层详解

### 2.1 交易状态筛选

- **剔除条件**：`is_suspended = True`（当日停牌）、`delisted_date` 距今 < 30 天（临近退市）、历史数据不足 20 个交易日
- **数据来源**：`get_symbols(sec_type1=1020, sec_type2=102001)` 返回的 `is_suspended`、`delisted_date` 字段
- **免费可用**：✅ 掘金免费函数
- **剔除原因**：停牌标的无法交易，退市标的将消失，数据不足的标的无法计算动量

### 2.2 流动性筛选

- **硬性门槛**：过去 20 个交易日日均成交额 > 5000 万元
- **数据来源**：`history(frequency='1d', fields='amount')` 获取每日成交额
- **免费可用**：✅ 掘金免费函数
- **剔除原因**：个人 100 万级别资金如果快进快出（特别是触发空仓风控时），必须有足够的流动性支撑。日均成交额低于 5000 万意味着单次买卖会产生巨大的冲击成本和滑点
- **规模代理**：A 股 ETF 的成交额与资产规模高度正相关。日均成交额不足 5000 万的 ETF，其 AUM 几乎不可能超过 5 亿。因此流动性筛选在实际效果上也充当了规模筛选的角色。如需严格按 AUM 过滤，可通过 `aum_dict` 参数注入外部数据。

### 2.3 波动率质量

- **硬性门槛**：
  - 近 20 个交易日年化波动率 < 40%（`日收益率 std × sqrt(252)`）
  - 无单日涨跌幅超过 15% 的异常跳空
- **数据来源**：`history(frequency='1d', fields='close')` 计算日收益率
- **免费可用**：✅ 掘金免费函数
- **剔除原因**：波动率过大的 ETF 说明底层资产极度不稳定（如杠杆 ETF、单行业主题 ETF 遭遇事件冲击），不适合作为动量交易的载体。此筛选替代了原设计中的折溢价率筛选——折溢价率的 IOPV 数据在免费 SDK 回测模式下不可用，而波动率质量可以用日线数据实现相同目的：过滤掉市场结构异常的标的。

### 2.4 大类资产去重

动量策略的核心竞争力来自"资产轮动"——在不同大类之间切换，而非在同一类资产内部追涨。若沪深300、上证50、沪深300成长这三只高度相关的 ETF 同时入选，动量排名时它们会因同涨同跌而霸占全部席位，导致组合承受三倍的系统性风险。

**去重规则**：同一大类资产只保留流动性最强（20 日均成交额最大）的一只。

**预设大类映射表**：

| 大类 | 说明 | 代表标的 |
|------|------|---------|
| A 股大盘 | 沪深 300、上证 50 | `SHSE.510300`, `SHSE.510050` |
| A 股中小盘 | 中证 500/1000/2000 | `SHSE.510500`, `SHSE.512100` |
| A 股创业科创 | 创业板、科创 50 | `SZSE.159915`, `SHSE.588000` |
| 红利防御 | 红利低波、中证红利 | `SHSE.515100`, `SHSE.510880` |
| 跨境对冲 | 纳指、标普 500 | `SHSE.513100`, `SHSE.513500` |
| 商品避险 | 黄金、有色 | `SHSE.518880`, `SHSE.518850` |

## 3. 数据可用性总览

| 筛选层 | 所需数据 | SDK 函数 | 免费 | 回测可用 |
|--------|---------|---------|:--:|:--:|
| 交易状态 | `is_suspended`, `delisted_date` | `get_symbols()` | ✅ | ✅ |
| 流动性 | 20 日日均成交额 | `history(fields='amount')` | ✅ | ✅ |
| 波动率质量 | 20 日日收益率序列 | `history(fields='close')` | ✅ | ✅ |
| 大类去重 | 无额外数据需求 | （内置映射表） | ✅ | ✅ |
| **可选**: 规模 | AUM / 资产净值 | `fnd_get_etf_constituents()` | ❌ 付费 | — |
| **可选**: 折溢价 | IOPV vs 现价 | `last_tick().iopv` | ⚠️ 实时 tick | ❌ |

**核心结论**：四层筛选中前三层（交易状态 + 流动性 + 波动率质量）+ 第四层去重，全部基于掘金免费 SDK 在回测和实盘中均可用的函数。AUM 和折溢价作为可选增强层，有外部数据时传入即可启用。

## 4. 管道对接策略的完整闭环

```
定时触发（每周一 09:31）
      │
      ▼
┌──────────────────────────────────────┐
│ Step A: 获取全市场 ETF 数据           │
│ get_symbol_infos(sec_type1=1020, ...) │ → 全市场 ETF 列表 (~1000 只)
│ get_symbols(...)                      │ → 交易状态 (is_suspended)
│ history(frequency='1d', ...)          │ → 30 日日线 OHLCV
└───────────────┬──────────────────────┘
                ▼
┌──────────────────────────────────────┐
│ Step B: EtfFilterPipeline 执行筛选    │
│ filter_and_select_seeds(...)          │
│   ._filter_trading_status()           │ → 剩余 ~700 只
│   ._filter_liquidity()                │ → 剩余 ~100 只
│   ._filter_volatility_quality()       │ → 剩余 ~80 只
│   ._dedup_by_asset_class()            │ → 种子池 5-8 只
└───────────────┬──────────────────────┘
                ▼
┌──────────────────────────────────────┐
│ Step C: 更新策略 ETF 池，重新订阅     │
│ context.etf_pool = pool               │
│ subscribe(symbols=pool, ...)          │
└───────────────┬──────────────────────┘
                ▼
┌──────────────────────────────────────┐
│ Step D: 每日动量检查 + 网格交易       │
│ EtfMomentumGridStrategy 状态机        │
│ order_target_value(...)               │
└──────────────────────────────────────┘
```

## 5. 可选数据注入

当你拥有外部数据源时（如 `akshare`、`tushare`、Wind 终端等），可通过以下方式启用额外筛选层：

```python
pipeline = EtfFilterPipeline()

# 方式 1: 注入 AUM 启用规模筛选
aum_data = {'SHSE.510300': 120_000_000_000, ...}
pool = pipeline.filter_and_select_seeds(
    etf_info_df=all_etfs,
    daily_bar_dict=daily_bars,
    aum_dict=aum_data,         # ← 传入则启用
)

# 方式 2: 注入折溢价率（实时模式）
premium_data = {'SHSE.510300': 0.002, ...}   # 0.2% 溢价
pool = pipeline.filter_and_select_seeds(
    etf_info_df=all_etfs,
    daily_bar_dict=daily_bars,
    premium_dict=premium_data,  # ← 传入则启用
)
```

## 6. 实战价值

1. **防止流动性黑天鹅** — A 股很多特定主题 ETF（低碳、传媒等）看似涨幅巨大、动量极强，但日均成交额可能只有几百万。通过强制流动性洗礼，100 万资金永远不会误入这些容易"买进去就出不来"的流动性死水。

2. **拒绝影子共振** — 通过大类资产分类去重，确保入选的标的在底层逻辑上天然对冲。这是资产轮动策略控制回撤的底层硬逻辑——不是靠止损，而是靠"不把鸡蛋放在同一个篮子里"。

3. **波动率质量保证** — 年化波动率 > 40% 的 ETF 往往是单边押注型标的（如单行业、杠杆产品），它们会破坏动量策略的稳定性。波动率质量筛选确保动量信号来自趋势而非噪音。

4. **完全基于免费数据** — 四层筛选中的每一层都使用掘金免费 SDK 可获取的数据，回测和实盘均可运行，不依赖付费接口或外部数据源。
